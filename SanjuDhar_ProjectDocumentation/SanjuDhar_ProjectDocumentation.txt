Event Management & Ticket Booking System
1. Project Overview
This project is a full-stack Event Management and Ticket Booking System that allows users to:
•	Register and log in
•	Browse events
•	Filter and sort events
•	Book tickets
•	Manage events (CRUD operations)
The system is divided into:
•	Backend (Node.js, Express, MongoDB)
•	Frontend (HTML,CSS,JS)
________________________________________
2. Backend Architecture
Folder Structure
Controllers/
middlewares/
models_/
routers/
server.js
.env
2.1 Controllers
Controllers handle the business logic:
•	event.js → Add, fetch, filter, and manage events
•	login.js → User authentication
•	userRegister.js → User registration
•	update.js → Update user 
•	delete.js → Delete user
2.2 Middlewares
Middlewares validate and process requests before reaching controllers:
•	event.mid.js → Event validation
•	login.mid.js → Login validation
•	user.mid.js → User validation
•	update.mid.js → Update validation
•	delete.mid.js → Delete validation
2.3 Models
Defines MongoDB schemas using Mongoose:
•	event.model.js → Event schema (Name, Category, Location, Event_Date,Starting_Time,Ending_Time,Price.)
•	user.model.js → User schema (first_name, last_name, email, password.)
2.4 Routers
Handles API routes:
•	event.router.js → Event-related APIs
•	login.router.js → Authentication routes
•	user.routers.js → User-related routes
•	update_user.router.js → Update operations
•	delete.router.js → Delete operations
________________________________________
3. API Endpoints
Auth APIs
•	POST /register → Register user
•	POST /login → Login user
Event APIs
•	POST /events → Create event
•	GET /events → Get all events
•	GET /events?category=&sort= → Filter/sort events
•	PUT /events/:id → Update event
•	DELETE /events/:id → Delete event
________________________________________
4. Frontend Overview
The frontend interacts with backend APIs and provides a user-friendly interface. Below is the detailed flow of what happens when a user clicks different options:
4.1 User Authentication Flow
Login page:-
 ●    If doesn’t have any account or you are not registered click on sign up
•	In sign up User enters First Name, Last Name, Email, Password
•	On clicking Sign up Button:
o	Frontend validates input fields
o	Sends POST /register request
o	On success → User is redirected to Login page
o	On error → Error message is displayed
Login Page
•	User enters Email and Password
•	On clicking Login Button:
o	Sends POST /login request
o	If credentials are correct → Redirect to Event Dashboard
o	If incorrect → Shows Error
________________________________________
4.2 Event Listing Page (Dashboard)
•	When user lands on dashboard:
o	GET /events API is called
o	All events are displayed in list/cards format
Filter Option
•	User selects Category (e.g., Music, Tech)
•	On change:
o	GET /events?category=value
o	Only filtered events are shown
Sort Option
•	User selects sorting (Price / Date)
•	On change:
o	GET /events?sort=value
o	Events are sorted accordingly
________________________________________
4.3 Add Event (Admin/User Action)
•	User can fills event form:
o	Event Name, Category, Location, Event_Date, Starting_Timed En-ing_Time, Price
•	On clicking Add Event Button:
o	Sends POST /events
o	On success → Event added and dashboard refreshes
o	On error → Error message shown
________________________________________
4.4 Delete Event
•	User clicks Delete Button on an event
•	Confirmation prompt appears
•	On confirm:
o	Sends DELETE /events/:id
o	Event is removed from UI
________________________________________
4.5 Ticket Booking Flow (If Implemented in UI)
•	User clicks Book Ticket on an event
•	Event details page opens
•	User selects quantity or checkout booking
•	On clicking Checkout:
o	Nothing will occur because any Bank account not connected.(This thing we will do in future).
